


Project A.

Descrição Rápida

Um sistema de gerenciamento, aonde qualquer empresa genérica pode usar de maneira
a gerenciar os seus funcionários e todos as demais necessidades que essa possa necessitar


Tecnologias:

React.js, como Front.
Postgresql, como Banco.



Back End: Node js + Graphql.


============ DOCUMENTAÇÃO, ISSUES.

- Toda alteração do sistema deve ter uma issue associada, com uma descrição
detalhada de todas as alterações a serem feitas, todo commit, deve ser
bem descrito com todas as alterações e atividades realizadas, deve ser realizado
alguma forma de manual a ser relacionado ao sistema de maneira que qualquer usuário possa
acionar esses manuais de maneira a aprender qualquer dificuldade sobre o que a página
e o processo atual se propõe a fazer.  


============ LOGIN, PERFIS, CARGOS

- Login, autenticação.
- Tipos de perfis de usuários:
    - Admin
    - Gerente
    - Funcionario
- Devem existir cargos, esses cargos, devem ser cadastrados pelo
gerente, sendo que os cargos são herança dos perfis, de maneira que um 
cargo só pode ser um dos tipo: Funcionário, gerente e Admin, não terão a noção
de cargos, e apenas o Admin pode gerar outros Admins ou um gerente.

============ LAYOUT, MOBILE, RESPOSIVIDADE 

- O layout do sistema deve ser totalmente resposívo, podendo, tendo uma versão
para telas de celular.

- O sistema deve ter a opção de tema escuro e claro, sendo essa alteração possível 
de ser alterada pelo admin, exclusivamente.

============ LINGUAGEM, BANCO  

- O sistema, deve salvar todos os termos e textos do sistema no banco, aonde
cada termo e qualquer texto usado no sistema deve ser traduzido por esse termo de acordo
com a linguagem que qualquer usuário pode alterar ao estar logado, toda a lógica 
com relação ao acesso ao banco por esses termos ainda será definida.
